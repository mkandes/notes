**Microsoft Presidio** is an open-source SDK designed to help organizations detect and anonymize Personally Identifiable Information (PII) in text and 
images.

It works as a two-step pipeline: **Analysis** (finding the PII) $\rightarrow$ **Anonymization** (masking or replacing it).

Here is a comprehensive guide on how to get started and use it.

---

### 1. Installation
Presidio consists of two main components: the **Analyzer** and the **Anonymizer**. You will also need a Natural Language Processing (NLP) model 
(Presidio uses SpaCy by default).

Run these commands in your terminal:

```bash
# Install the Presidio libraries
pip install presidio-analyzer presidio-anonymizer

# Download the English NLP model for SpaCy
python -m spacy download en_core_web_lg
```

---

### 2. Basic Workflow (The "Hello World")
To use Presidio, you create an instance of the `AnalyzerEngine` to find the PII and then pass those results to the `AnonymizerEngine`.

#### Complete Code Example:
```python
from presidio_analyzer import AnalyzerEngine
from presidio_anonymizer import AnonymizerEngine

# 1. Initialize the engines
analyzer = AnalyzerEngine()
anonymizer = AnonymizerEngine()

# The text we want to scrub
text_to_analyze = "My name is John Doe and my phone number is 202-555-0123. I live in New York."

# 2. Analysis Phase: Detect PII
# This returns a list of recognized entities (type, start/end index, score)
analysis_results = analyzer.analyze(text=text_to_analyze, entities=[], language='en')

print("Detected PII:", analysis_results)

# 3. Anonymization Phase: Mask the detected PII
anonymized_result = anonymizer.anonymize(
    text=text_to_analyze,
    analyzer_results=analysis_results
)

print("Anonymized Text:", anonymized_result.text)
```

**Output:**
> **Detected PII:** `[RecognizerResult(person=..., phone_number=..., location=...)]`  
> **Anonymized Text:** `My name is <PERSON> and my phone number is <PHONE_NUMBER>. I live in <LOCATION>.`

---

### 3. Deep Dive: The Analyzer
The `AnalyzerEngine` uses a combination of **Regular Expressions (Regex)**, **Checksums**, and **NLP Models** to find PII.

*   **Filtering Entities:** You can tell Presidio exactly what to look for to avoid false positives.
    ```python
    # Only look for emails and credit cards, ignore names/locations
    results = analyzer.analyze(text=text, entities=["EMAIL_ADDRESS", "CREDIT_CARD"], language='en')
    ```
*   **Confidence Threshold:** You can set a minimum score (0.0 to 1.0) to filter out low-confidence detections.
    ```python
    results = analyzer.analyze(text=text, score_threshold=0.4)
    ```

---

### 4. Deep Dive: The Anonymizer
The `AnonymizerEngine` is flexible. You don't have to just use `<PERSON>`; you can replace data with hashes, fake data, or completely remove it.

#### Customizing the Output (Operators):
You can define how different entities are handled using `Operator` objects.

```python
from presidio_anonymizer.entities import Operator

# Define a custom mapping for replacements
operators = {
    "PERSON": Operator.replace(text="USER"), 
    "PHONE_NUMBER": Operator.mask(masking_char="*", chars_to_mask=10, proportion=0.8),
    "LOCATION": Operator.hash() # Hashes the location for data analysis purposes
}

anonymized_result = anonymizer.anonymize(
    text=text_to_analyze,
    analyzer_results=analysis_results,
    operators=operators
)
```

---

### 5. Advanced Usage: Custom Recognizers
If you have company-specific PII (like a custom Employee ID format), you can create a **PatternRecognizer**.

```python
from presidio_analyzer import PatternRecognizer, Pattern

# Define a regex for an Employee ID (e.g., EMP-12345)
emp_id_pattern = Pattern(r"\bEMP-\d{5}\b", name="employee_id")
emp_recognizer = PatternRecognizer(supported_entity="EMPLOYEE_ID", patterns=[emp_id_pattern])

# Add the custom recognizer to the engine
analyzer.registry.add_recognizer(emp_recognizer)

text = "The employee ID is EMP-99876"
results = analyzer.analyze(text=text, language='en') # Now it finds EMPLOYEE_ID
```

---

### 6. Deployment Options
While the SDK (shown above) is great for scripts and apps, Presidio can also be deployed as a **REST API**.

1.  **Docker:** Microsoft provides official Docker images for both the `presidio-analyzer` and `presidio-anonymizer`.
2.  **Architecture:** In a production environment, you typically run these as two separate microservices. Your application sends text to the Analyzer 
$\rightarrow$ receives coordinates $\rightarrow$ sends coordinates to Anonymizer $\rightarrow$ receives scrubbed text.

### Summary Table: Presidio at a Glance

| Component | Purpose | Primary Tooling |
| :--- | :--- | :--- |
| **Analyzer** | Detects PII (What and Where) | SpaCy, Regex, Logic Rules |
| **Anonymizer** | Modifies the text (How to hide) | Masking, Hashing, Replacing |
| **Registry** | Manages recognizers | `analyzer.registry` |
| **Operators** | Defines replacement logic | `Operator.replace`, `Operator.mask` |
