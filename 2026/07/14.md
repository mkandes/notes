# How to run Ollama on Expanse (within a JupyterLab session)

The easiest and most secure way to launch a JupyterLab session on Expanse is via [galyleo](https://github.com/sdsc/galyleo). Here, let's start by creating a a very basic conda envrionment file named `jupyter.yaml`.

```
name: jupyter

channels:
  - conda-forge

dependencies:
  - jupyterlab
```

Next, put `galyleo` in your `PATH`.

```
export PATH="/cm/shared/apps/sdsc/galyleo:${PATH}"
```

Then launch the JupyterLab session within the conda environment on a GPU node within the `gpu-debug` partition. 

```
galyleo launch --account ddp324 --partition gpu-debug --cpus 10 --memory 92 --gpus 1 --time-limit 00:30:00 --conda-yml jupyter.yaml --quiet
```

Copy and paste the URL provided into your web broswer and wait for the job to start up.

```
https://sureness-careless-coral.expanse-user-content.sdsc.edu/?token=XXXXXXXXXXXXXXXXXXXXXX
```

```

          52032399 gpu-debug galyleo-  mkandes  R       0:36      1 exp-7-59

```

Once the JupyterLab session begins, open its Terminal app within the Launcher window. You now have a basic conda environment where you can iterate on building any software environment. e.g., your Presidio environment. Unfortuantely, right now it looks like the latest Ollama distributions from `conda-forge` are effectively broken and do not properly support NVIDIA GPUs out of the box. There may be some explaination for this in the maintainers changelog, but we can also simply download a fully functional Ollama distribution from the project itself. 

First, create a software directory in your `$HOME` directory to store Ollama.

```
mkdir -p ~/software/ollama
```

Next, change directories to this location.

```
cd ~/software/ollama
```

Then download the latest Ollama distribution via `wget`.

```
wget https://github.com/ollama/ollama/releases/download/v0.32.0/ollama-linux-amd64.tar.zst
```

Extract the compressed archive file.

```
tar --use-compress-program=unzstd -xvf ollama-linux-amd64.tar.zst
```
